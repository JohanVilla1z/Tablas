Tabla con las diferencias entre Git Local y Git Remoto

Git Local Git Remoto
  • Git local funciona alojado en nuestro computador sin conexion a internet
  • Es un sistema de control de versiones para proyectos propios
  • Puedes crear, modificar y confirmar cambios en tu repositorio local sin necesidad de conexión a Internet
  • Los cambios en el repositorio local no afectan a otros colaboradores hasta que los envíes al repositorio remoto
  • Los cambios realizados a los proyectos localmente se ven guardados inmediatamente
    		•
  • Git remoto funciona completamente alojado en la nube
  • Es una red social del código, una hoja de vida, para mostrar los repositorios
  • Puedes tener varios repositorios remotos, cada uno con diferentes permisos (lectura o lectura y escritura)
  • Colaborar con otros implica gestionar estos repositorios remotos, enviando y trayendo datos de ellos
  • Puedes ver las diferencias entre la versión local y la remota antes de fusionar cambios
